# Contributing to openMAG

Thank you for your interest in contributing to the openMAG Project.

The openMAG Project is an educational, research-focused open-source project. Contributions are welcome
as long as they align with the project's goals of safety, legality, and technical clarity.

## What we welcome

- Documentation improvements (clarity, structure, accuracy)
- Technical corrections supported by evidence (logs, photos, datasheets)
- Research notes clearly labeled as hypothesis or confirmed behavior
- Troubleshooting cases with reproducible context
- Diagrams and illustrations that improve understanding

## What we do not accept

- Proprietary firmware or binaries
- Leaked or vendor-restricted tools
- NDA-protected information
- Instructions intended to enable unlawful activity

## How to contribute

1. Fork the repository
2. Create a branch for your change
3. Make focused, well-described commits
4. Open a Pull Request explaining:
   - What changed
   - Why it changed
   - Any evidence or references

## Documentation guidelines

- Write for a technically curious but non-expert reader
- Separate concepts from procedures
- Avoid unnecessary jargon; define terms when first used
- Do not add decorative icons or emojis

By contributing, you agree that your work will be released under the project license.
